Introduction

Welcome to the Satellite Orbit and Event Predictor (SOEP) project plan. This project aims to develop a C++ application that predicts satellite passes, collision risks, orbital decay, and re-entry events.

Project Goal

SOEP provides real-time data, visualizations, and an interface for satellite tracking and space object management. The application is designed to assist in monitoring satellite activities and predicting significant events related to satellite orbits.

Frontend Developers

The frontend team focuses on the user interface and user experience, implementing the components that users interact with directly.

Math Team

This team is responsible for mathematical algorithms, data processing, and backend integration. Members contribute to both mathematical computations and some backend functions.

Backend Developers

The backend team works together with the math team. Their primary objectives are to handle the server-side logic, database management, and API integration, ensuring the frontend team receives the necessary data and functionality.

Project Plan

  1. Project Initialization & Research: Understand the problem deeply and set a strong foundation for the project. Research satellite tracking algorithms, collision prediction models, and relevant data sources.
  2. System Design & Architecture: Set up a simple API node and a backend to process the data, and also a simple frontend that receives all the information.
  3. Algorithm Development & Math: Develop and test core algorithms for satellite pass prediction, collision probability, orbital decay, and re-entry, ensuring the math is correct.
  4. Visualization Module Development: Develop a visualization module to allow users to view satellite orbits and real-time tracking.
  5. Interface (UI) Development: Create an interactive, user-friendly interface for data input and result visualization.
  6. System Integration: Integrate all components (algorithms, visualizations, and UI) into a unified system.
  7. System Testing: Perform thorough functional, non-functional, and performance testing on the fully integrated system. Identify and resolve bugs or usability issues.

API

We will use the N2YO API to get the satellite information.

Backend

The backend works as both the API and for serving the frontend through server-side rendering. We will use templating to send back parts of HTML. When in need of C++ functions, we will be able to either call on the executable to generate some file or, if the C++ code is built into a DLL, call the DLL functions and receive the data back that way. The whole backend will be containerized containing the SOEP executable or dynamic library and will work as a middleman between those two parts of our program.

The technology that will be used is:

  • Node.js
  • Express
  • Some templating engine (Pug, Handlebars, etc.)
  • FFI for using DLLs (optional)
  • Child Processes library for calling executables (optional)
  • Postgres library for storing the results of our simulations if we want to store the data in a database instead of blob storage

The optional tag indicates technology that will be used depending on our chosen approach. For example, if we decide to use DLLs instead of executables, we will use FFI and not Child Processes.

The backend will also be containerized using Docker, so that we can spin up the stack whenever and wherever we need.

C++

This component will handle all calculations for satellite orbits, collision probability, and predictions for when a chosen satellite will pass over a specific geographic location. In the future, we will add other calculations.

Frontend

The frontend will focus on visualization and user interface. The website will display all the information from the satellites that are being tracked and aims to be user-friendly and modern, with a strong focus on functionality.

The website will be developed using HTMX, CSS, and JavaScript. It will display a 3D rendering of the Earth with all current satellites visible at their current positions. When a user clicks on a satellite, its information will show in a popup, and its orbit will be visible around the Earth. WebGL will be used to render the 3D Earth.

Detailed 3-Week Plan

We will have two meetings per week, every Tuesday and Friday, where we discuss the status of the project and ensure everybody knows the next steps.

Week 1

In the first week, we'll focus on getting the project off the ground. We'll start by meeting up to discuss the project goals and make sure everyone is aligned. We'll dive into researching satellite tracking algorithms and collision prediction models, and we'll familiarize ourselves with the N2YO API to fetch satellite data. We've distributed tasks among the team, and everyone has an issue to work on. We'll set up our development environment, including repositories and any necessary software. We'll also decide on our communication methods and schedule regular meetings to keep things running smoothly. By the end of the week, we should have a detailed plan and be ready to dive into the next steps.

Week 2

In Week 2, we will continue developing the project by integrating the N2YO API and working on backend functionality for satellite predictions and collision risks. The frontend will focus on building the user interface and initial visualizations, including basic 3D rendering with WebGL. We'll also ensure data flow between components. Documentation will be updated, and progress will be reviewed during our Tuesday and Friday meetings. By the end of Week 2, we should have a very basic version of the application running.

Week 3

In Week 3, we'll focus on reviewing the progress we've made and adjusting our plans as needed. We'll assess the performance of the key features we've developed so far, check for any issues, and make improvements. This includes reviewing both the technical aspects and how well the team has been collaborating.

While we conduct this review, we'll also continue working on ongoing tasks to avoid losing momentum. This will ensure that development doesn't slow down as we evaluate our progress.

We'll reassign roles and update tasks based on the current status of the project, ensuring everyone is focused on the most important priorities. We'll also update the roadmap and set clear goals for the upcoming weeks, keeping the team aligned and on track.

By the end of the week, we'll have a refined plan, updated tasks, and a clearer direction, while keeping our existing work moving forward.
