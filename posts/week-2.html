Week 2 - Progress Update

What We Achieved This Week

This is the second week of our Satellite Orbit and Event Predictor (SOEP) project, and we made good progress in building the foundation. Here’s a summary of what we accomplished:

  • API Integration: We successfully made our first basic API call to retrieve satellite data, which will be essential for the tracking functionality.
  • Website Design: We implemented a basic design for the website, laying the groundwork for the user interface.
  • Frontend Development: We started implementing key components for displaying satellite information on the website.
  • Cloud Setup: Initially, we set up Google Cloud services for our infrastructure, but later decided to change this approach.
  • Concurrency Decisions: We discussed and evaluated different data handling techniques, debating between using DLL, file-based, or alternative methods for better performance.
  • Multithreading Implementation: To optimize performance, we implemented multithreading for handling satellite data calculations.
  • Blog Setup: We created a blog section to document our progress and share updates.
  • CUDA vs. CPU: The team began investigating whether to leverage CUDA for GPU acceleration or stick with CPU-based processing for satellite calculations.
